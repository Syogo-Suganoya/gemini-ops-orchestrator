# 🚀 プロジェクト名：Legacy-Ghost（レガシー・ゴースト）

![alt text](image.png)

**〜 属人化を解消する自律型「退職・引き継ぎゴースト」エージェント 〜**

## 💡 設定した課題とその背景

企業の成長において、業務の「属人化」と担当者の離職に伴う「引き継ぎ漏れ」は極めて深刻な課題です。従来の引き継ぎ業務は、退職予定者が業務の合間に手動で資料を作成するため、時間的猶予のなさから内容が形骸化しがちでした。

その結果、前任者が「メールで交わした細かい約束」「Meetでの口頭の合意」「顧客ごとのデリケートな傾向や性格」といった暗黙知（デジタル足跡）が後任に共有されず、担当者変更をきっかけとした顧客満足度の低下や失注のリスクが常に発生しています。

## 🤖 AIエージェントの自律的な振る舞い（価値の中心）

本プロダクトは、退職予定者のデジタル足跡を自律的に全スキャンし、**前任者の「仕事の魂」を憑依（イタコ）させたかのような超高解像度の引き継ぎ書を自動生成、後任のアサインからミーティング調整までを完全自動化するエージェント**です。

1. **Salesforceのステータス変更を起点とした自律起動**
SalesforceのUserオブジェクトを常時監視。該当社員のステータスが「退職予定」または「休職」に変更されたことをWebhookで検知した瞬間、人間の指示を待つことなく自律的に全引き継ぎパイプラインを起動します。
2. **デジタル足跡の網羅的スキャンと「仕事の魂」の抽出**
前任者が社内に遺したあらゆるデジタルアクティビティを裏側で一括収集（スキャン）します。
* **Gmail API：** 過去数ヶ月におよぶ顧客との全メール履歴から、文脈や連絡の「癖」を抽出。
* **Google Meet（Google ドライブ）：** 録画データや自動生成された議事録から、直近の商談での口頭の約束事を分析。
* **Google アナリティクス：** 前任者が管理していたマーケティングやプロダクトの成果数値を集約。


3. **Google Docs APIと連動した「超高解像度引き継ぎ書」の自律生成**
収集した莫大なコンテキストをベースに、Geminiが「顧客ごとの現在のステータス」「未解決の課題」に加え、通常は言語化されない「相手のキーマンの性格や、好む連絡頻度・トーン」までを自律推論。Googleドキュメント上に構造化された完璧な引き継ぎ書を自動生成します。
4. **後任へのパスとカレンダーへの自動ねじ込み**
Salesforce上で自動選定された後任担当者の**Google カレンダー**の空き時間をエージェントが自律的にチェック。最短の最適な枠に、生成した引き継ぎドキュメントのリンクを内包した「〇〇社 業務引き継ぎミーティング」という予定を自動でセット（ねじ込み）し、スムーズな業務バトンタッチを強制執行します。

## ⚙️ 技術スタックと選定理由（実装力）

大量の非構造化データ（音声・テキスト・数値）を一括処理する、Geminiの「長尺コンテキスト」の強みを最大化させたアーキテクチャです。

* **実行環境：GKE (Google Kubernetes Engine) / Cloud Run**
* *選定理由:* SalesforceからのWebhookをトリガーに、退職者の数ギガバイトにおよぶメール履歴や動画議事録データを、並列かつ高速に収集・パイプライン処理するための強力な実行基盤としてGKEおよびCloud Runを採用。


* **AI・推論基盤：Gemini 1.5 Pro (Vertex AI)**
* *選定理由:* 本プロダクトは、Gemini 1.5 Proの最大の特徴である「200万トークンの超長尺コンテキスト窓」の真骨頂です。数ヶ月分のメールやミーティング動画という膨大な非構造化データを一括で読み込ませ、破綻することなく一つの高精度な引き継ぎ文脈へと昇華（推論）させるために不可欠な選定です。


* **Googleサービス連携：Gmail API, Google Meet (Drive) API, Google Analytics Data API, Google Docs API, Calendar API**
* *選定理由:* 企業活動のインフラであるGoogle Workspaceのデータを全方位から安全に取得・解析し、最終的なアウトプットのドキュメント生成やカレンダー調整までを一気通貫で自動制御するため。
