# 🚀 プロジェクト名：Trend-Launcher（トレンド・ランチャー）

![alt text](image-1.png)

**〜 アナリティクス連動型「自律型・新商品ローンチ」エージェント 〜**

## 💡 設定した課題とその背景

従来のビジネスにおいて、市場のニーズを調査し、それに応じた新商品を企画・開発し、さらに営業現場や会計システムへ展開するまでには、膨大な時間と組織間（マーケティング・企画・営業・経理）の調整コストが発生していました。

このプロセスにおけるタイムラグにより、Webサイトや広告を通じて顧客からリアルタイムに寄せられている「今まさに急上昇している潜在ニーズ（未充足ニーズ）」を捉えきれず、競合他社に先を越されてしまうという重大な機会損失が発生しています。

## 🤖 AIエージェントの自律的な振る舞い（価値の中心）

本プロダクトは、人間が市場調査をして商品開発を行う従来のサイクルを根底から変える存在です。**Webサイト上のデータから「顧客の飢餓感」を自律的に察知し、人間の承認を待たずに商品の企画・マスター登録・会計設定・営業へのアプローチ指示までを1時間単位で勝手に執行する超速の「自律型事業開発」エージェント**です。

1. **Google アナリティクスによる「未充足ニーズ」の自律検知**
Cloud Schedulerを起点に、**Google Analytics Data API**を定期的に自動巡回。サイト内検索キーワードや特定ページの滞在時間、直帰率などのデータを複合解析し、「検索数が急上昇しているにもかかわらず、自社サイトに対応するコンテンツや商品が存在しないキーワード（顧客の未充足ニーズ）」を自律的に抽出します。
2. **Gemini 1.5 Proによる「商品企画・仕様定義」の自律生成**
抽出されたキーワード（例：『〇〇機能 対応』など）を基に、Gemini 1.5 Proが即座に商品企画を開始。「このニーズを最速で満たす新商品の名称、想定価格、製品仕様、そしてバックオフィスに即時連携するための会計用勘定科目（売上区分）」までを、一貫したビジネスロジックに基づいて自律生成します。
3. **Salesforceおよび会計システムへの「マスター自動登録」**
生成された製品データを、Salesforce APIを介して「Product2（商品）」オブジェクトへ即座に自動登録。同時に価格表（PricebookEntry）も自動で作成し、商談で即座に選べる状態を整えます。さらに、連携されたGoogleスプレッドシート（または会計API）に、新しい商品コードと仕訳ルールを自律的に書き込み、バックオフィスの準備も並行して完了させます。
4. **営業チームへの「自動ローンチ通知」と見込み顧客の自動アサイン**
**Firebase Cloud Messaging (FCM)** を活用し、営業チームのモバイル端末（Flutterアプリ）やSlackへ「【自動ローンチ】新商品『〇〇』を製品カタログに追加しました」とプッシュ通知を送信。同時に、Salesforce内から「過去にそのキーワードに類似する問い合わせをしてきた見込み顧客（リード）」をエージェントが自動で抽出し、ターゲットリストとして営業へ即座に手渡します。

## ⚙️ 技術スタックと選定理由（実装力）

1時間単位の高速なパトロールと、エンタープライズのCRM・ERPを自律的に書き換える強固な連携アーキテクチャです。

* **実行環境：Cloud Run + Cloud Scheduler**
* *選定理由:* 1時間に1回の定期パトロール（Cloud Scheduler）の実行と、ニーズ検知時に発生する「商品企画・Salesforceや Sheets への一斉API書き込み」という高負荷なバッチ処理を、サーバーレスで高速かつ低コストに実行するため。


* **AI・思考基盤：Gemini 1.5 Pro (Vertex AI)**
* *選定理由:* 単なるキーワードの抽出にとどまらず、「市場価格の適正推論」や「組織の既存カタログと競合しない製品仕様の策定」、さらには「正しい勘定科目の割り当て」といった、ビジネスドメインを横断する高度なコンテキスト（文脈）理解と推論を行わせるため。


* **周辺ビジネスシステム連携：Salesforce API & Google Sheets API**
* *選定理由:* 企画された製品を即座に「売れる状態」にするフロント（Salesforce）と、バックオフィス（スプレッドシート/会計システム）の双方のマスターデータを、人間の仲介なしに完全同期で自動起票するため。


* **プッシュ通知・フロントUI：Firebase Cloud Messaging (FCM) & Flutter**
* *選定理由:* 新商品の自動追加という企業の超重要トピックを、外出先の営業マンのモバイル端末へ遅延なく確実に届けるため（FCM）。また、アサインされた見込み顧客リストをスマートフォンの画面からワンタップで確認・架電できる滑らかな営業動線をFlutterで構築するため。
