# 🚀 プロジェクト名：Legacy-Agent（レガシー・エージェント）

**〜 ユーザーの意思と言動を学習する自律型「デジタル魂継承」エージェント 〜**

## 💡 設定した課題とその背景

デジタル社会において、個人が遺すデジタル資産（ドキュメント、アカウント情報、大切な人へのメッセージなど）の管理は「終活」における大きな課題となっています。万が一の事態が起きた際、遺族はどのデータが重要なのか判別できず、また故人の「本当の想い」や「手続きの意思」を正確に知る術がありません。

これらは極めてデリケートかつプライベートな情報であるため、従来のクラウドストレージにただ保存しておくだけでは、適切なタイミングでの適切な人への引き継ぎや、個人の尊厳を守った情報開示が不可能であるという課題があります。

## 🤖 AIエージェントの自律的な振る舞い（価値の中心）

本プロダクトは、単なるデータのバックアップツールや遺言書の保管サービスではありません。**ユーザーの日常の活動を静かに見守り、万が一の事態を自律的に検知した瞬間、個人の「人格」と「意思」をエミュレートして重要情報の整理やメッセージ伝達を自律的に代行するエージェント**です。

1. **生存確認（死活監視）の自律実行とフェーズ移行**
一定期間ユーザーのアカウントログインや活動がない場合、または特定のカレンダー予定（通院や体調不良の記録など）を起点に、エージェントが自ら「体調は大丈夫ですか？」とプッシュ確認を実行。一定期間応答がない場合に、緊急事態と判断して「デジタル意思代行フェーズ」へ自律的に移行します。
2. **散らばったドキュメントからの「重要情報」の自律選別**
ユーザーが遺した膨大なデータ（Google ドライブ内の書類、メモ、メールなど）を巡回し、遺族や弁護士に「本当に伝えるべき重要情報（口座情報、契約関係、遺言に関連する書類）」だけをGeminiが自律的に選別・構造化して整理します。
3. **過去のデータに基づく「人格・感情」のエミュレート**
ユーザーが過去に書いた日記、メールの文面、発言の癖などの長期コンテキストを学習。万が一の際、遺された家族や関係者に対して、まるで本人が語りかけているかのような、その人らしい思考のニュアンスやトーンを再現したメッセージを自律生成します。
4. **カスタムボイスによる感情再現メッセージの届出**
生成されたメッセージを、**Text-to-Speech API**を用いてユーザー本人の声の音響特徴に近いカスタムボイスへと変換。冷たいテキストではなく、温かみのある「声」のリソースとして、指定された連絡先（Gmail等）へ自律的に届けます。

## ⚙️ 技術スタックと選定理由（実装力）

強固な死活監視トリガーと、個人のパーソナルデータを極めて高度に、そして安全に処理するためのアーキテクチャです。

* **実行環境：Cloud Run + Cloud Scheduler**
* *選定理由:* 24時間365日の定期的な死活監視（Cloud Schedulerによる定期実行トリガー）と、緊急フェーズ移行時の大規模なデータ解析・メッセージ生成処理を、セキュアかつサーバーレスに実行するため。


* **AI・検索基盤：Gemini 1.5 Pro & Vertex AI Search**
* *選定理由:* ユーザーが過去に遺した膨大なライフログ（テキスト、日記、メール）という超長尺の文脈を完全に理解し、その人ならではの「人格」をエミュレートするために200万トークンの文脈窓を持つGemini 1.5 Proを採用。また、散らばった個人データから必要な情報だけを即座にセマンティック抽出するためにVertex AI Searchを連携。


* **音声合成：Text-to-Speech API (Custom Voice)**
* *選定理由:* テキストを超えた「感情の再現」と「遺族への心理的ケア」を実現するため、本人の音声ニュアンスに極限までアジャストした合成音声を生成するため。


* **監視トリガー：Gmail API / Google Calendar API**
* *選定理由:* ユーザーの日常的なアクティビティ（メールの送受信、カレンダーの更新履歴など）の有無を、最も身近なインフラから安全にパトロールするため。
